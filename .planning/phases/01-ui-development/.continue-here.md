## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| userEvent v14 async | `userEvent.click` in v14 returns a Promise and must be awaited. Wrapping it in `act()` synchronously does not resolve the click. Tests using `act(() => { userEvent.click(...) })` will silently not fire the click. | advisory | Use `fireEvent.click` from `@testing-library/react` for synchronous click events in fake-timer tests. Alternatively, make test async and `await userEvent.click(...)` inside `act`. |
