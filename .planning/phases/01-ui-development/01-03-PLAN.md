## Phase Goal

**As a** podcast host, **I want to** enter the room with my co-host, press a single red record button, see a 3-2-1 countdown, watch a live timer, then stop and see the upload -> mix -> ready transitions, **so that** the recording experience is visually proven end-to-end (without yet wiring real audio).

<objective>
Implement the full Room view including the local state machine that Phase 3 will later drive from WebRTC signals (D-11). The state machine covers all six states: `idle -> countdown -> recording -> uploading -> mixing -> ready`. Build the standalone `RecordButton` and `MicVisualizer` components. Replace the `/room` placeholder.

Purpose: Lock the visual + behavioral contract of the recording flow. Phase 3 swaps the local click trigger for a peer signal without touching the state machine, and Phase 4 swaps the placeholder uploading/mixing transitions for real uploads. The state-machine boundary is the integration handoff (D-11).
Output: A fully animated room with working countdown, recording pulse, timer, state-label updates, and post-recording transitions that end on the `ready` state.
</objective>

State machine (from PATTERNS.md):
```
type RoomState = 'idle' | 'countdown' | 'recording' | 'uploading' | 'mixing' | 'ready'
```

State transitions (mirror HTML applyRoomState + onRecClick + runCountdown):
- idle  --(rec click)-->  countdown
- countdown (3 -> 2 -> 1 -> GO, then auto)  -->  recording
- recording  --(rec click again)-->  uploading
- uploading  --(setTimeout ~1500ms placeholder)-->  mixing
- mixing  --(setTimeout ~1500ms placeholder)-->  ready
- ready  --(rec click)-->  idle (restart)

While in `recording`: a setInterval increments a `seconds` counter, formatted via `formatTime(s) = HH:MM:SS`. Clear interval when leaving `recording`. Clear all timers on unmount (HTML clearRoomTimers, lines 719-728).

CSS class composition on the outer `<div className="room ...">`:
- 'recording' when state === 'recording'
- 'uploading' when state === 'uploading'
- 'mixing' when state === 'mixing'
- 'ready' when state === 'ready'
- (no class for idle and countdown; countdown overlay is its own .countdown element with .show modifier)

Countdown overlay: `<div className={'countdown' + (state === 'countdown' ? ' show' : '')}><div key={cdKey} className="num">{cdNum}</div></div>`. Increment `cdKey` on each countdown start so the `cdPop` CSS animation re-fires (RESEARCH Pitfall 4 — countdown animation restart). The displayed number transitions 3 -> 2 -> 1 -> 'GO' over four 1-second steps, then the state moves to `recording`.
